# Car Price Prediction Project

This project aims to predict car prices using machine learning techniques. It leverages a dataset of car specifications and features to train a predictive model, enabling users to estimate the market value of vehicles.

---

## Project Structure

The project is organized as follows:

* **`.venv/`** : Python virtual environment containing project dependencies.
* **`data/`** : Contains the dataset and processed data.
* **`raw/`** : Original dataset (`vehicles.csv`).
* **`processed/`** : Cleaned and preprocessed data (`cleaned_car_csv.csv`).
* **`splited/`** : Splitted data for train and test dataset.
* **`logs/`** : Log files for tracking application activity (`app.log`).
* **`models/`** : Saved trained models and related artifacts.
* `encoder.pkl`: Label encoder for categorical features.
* `model.pth`: Trained machine learning model.
* `scaler_x.pkl`: Scaler for input features (X).
* `scaler_y.pkl`: Scaler for target variable (y).
* **`notebooks/`** : Jupyter notebooks used for data exploration, preprocessing, and model development.
* **`outputs/`** : Contains any output generated by the project, such as plots or reports.
* **`src/`** : Source code for the application.
* `app.py`: Main application file (e.g., Flask or Streamlit app).
* `config.py`: Configuration settings for the application.
* `data_handling.py`: Functions for data loading and saving.
* `data_preprocessing.py`: Functions for data cleaning, transformation, and feature engineering.
* `logger.py`: Custom logging setup.
* `model_evaluation.py`: Functions for evaluating model performance.
* `model.py`: Contains the machine learning model definition and training logic.
* `prediction.py`: Functions for making predictions using the trained model.
* `test_app.py`: Unit tests for the application.
* **`app.py`** : A secondary app file (if applicable).
* **`learning.txt`** : Any additional notes or learning points during the project.
* **`README.md`** : This file (project documentation).
* **`requirements.txt`** : List of project dependencies.

---

## Data

The dataset used in this project is `vehicles.csv`, which contains information about various cars, including their make, model, year, mileage, and price. The data preprocessing steps involve cleaning the data, handling missing values, and potentially creating new features to improve model performance.

## Model

The project initially experimented with multiple machine learning models such as Linear Regression, Random Forest, Gradient Boosting, and XGBoost. After evaluating the performance of these models, we finalized a **Neural Network** model as the best-performing one. The model was trained using the dataset, and its performance was evaluated on a separate test set to assess its accuracy and generalization ability.
